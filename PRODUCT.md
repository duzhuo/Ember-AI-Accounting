# Ember AI 会计凭证 Agent — 产品说明

## 一、解决什么问题

企业财务人员每天需要将发票、合同、费用单据"翻译"成会计分录，再手工录入 SAP 系统。这个环节：

- **耗时**：一张发票需查科目表、核税码、计算含税金额，熟练工也需 5–10 分钟
- **易错**：税率填错、借贷方向颠倒、科目代码输错是常见问题，月末对账时才发现
- **依赖经验**：新员工需数月培训才能独立完成记账，离职风险高
- **批量瓶颈**：月末集中大量凭证，人工处理速度跟不上业务节奏

Ember AI 通过 AI Agent 将上述环节自动化：**用自然语言描述或上传单据，30 秒内生成可直接导入 SAP 的标准凭证。**

## 二、基于的会计准则

本系统的科目体系和凭证结构基于以下准则框架：

### 中国企业会计准则（CAS）

遵循财政部发布的《企业会计准则》及《企业会计准则——应用指南》（科目编码体系），核心科目映射如下：

| 科目代码 | 科目名称 | 对应准则 |
|---------|---------|---------|
| 112200 | 应收账款 | CAS 22 — 金融工具确认与计量 |
| 600101 | 主营业务收入-软件服务 | CAS 14 — 收入 |
| 600102 | 主营业务收入-商品销售 | CAS 14 — 收入 |
| 22210105 | 应交税费-应交增值税-销项税额 | CAS 依据增值税会计处理规定（财会[2016]22号） |
| 22210101 | 应交税费-应交增值税-进项税额 | 同上 |
| 660201 | 管理费用-招待费 | CAS 9 — 职工薪酬 / 费用类准则 |
| 220201 | 应付账款-员工报销 | CAS 22 — 金融工具 |

### 增值税处理

遵循《增值税会计处理规定》（财会[2016]22号），税码规则：

| 税率 | SAP 税码 | 说明 |
|-----|---------|-----|
| 13% | X1 | 一般货物、软件产品 |
| 6% | X6 | 服务类（现代服务业） |
| 0% | X0 | 免税 / 出口 |

### SAP 凭证类型

| 类型代码 | 含义 |
|---------|-----|
| DR | 客户发票（Debtor Invoice） |
| KR | 供应商发票（Vendor Invoice） |
| SA | 一般过账（General Ledger） |

> **注意**：本系统的科目代码为示例配置，实际使用时财务团队可通过管理界面自定义科目编码、税码映射和凭证类型，以匹配企业自身的 SAP Chart of Accounts。

## 三、如何帮助 SAP 客户

### 场景 1：自然语言 → 凭证

```
财务人员输入：
"销售软件产品给华为公司，不含税金额 100,000 元，税率 13%"

Agent 输出（30 秒内）：
行1  借  应收账款（112200）          113,000.00 CNY
行2  贷  主营业务收入-软件服务（600101）100,000.00 CNY
行3  贷  应交税费-应交增值税-销项税额   13,000.00 CNY
```

### 场景 2：发票图片 / PDF → 凭证

拍摄或上传增值税专用发票，OCR Agent 自动提取：购买方、销售方、金额、税率、发票号，生成凭证草稿，无需手工录入任何字段。

### 场景 3：批量上传 → SAP CSV 导入

拖入多张发票，系统逐一处理，全部完成后一键下载符合 SAP 批量导入格式的 CSV 文件（BUKRS/BLART/BLDAT/HKONT/WRBTR 等标准字段），直接在 SAP 中通过 FB01 / LSMW / BAPI 导入，无需二次录入。

### 场景 4：审批流

凭证生成后可提交指定审批人，状态机为：

```
草稿 → 待审批 → 已过账（通过）
              → 草稿（驳回，修改后重提）
```

全流程留存审计日志，满足内控合规要求。

## 四、能产生什么效益

| 维度 | 传统方式 | 使用 Ember AI | 提升幅度 |
|-----|---------|--------------|---------|
| 单凭证处理时间 | 5–10 分钟 | 30 秒以内 | **>90% 时间节省** |
| 批量处理能力 | 人工逐条 | 拖入多文件自动串行处理 | 无瓶颈 |
| 录入错误率 | 依赖人工经验 | 规则引擎保证借贷平衡、税额精确 | 显著降低 |
| 新员工上手时间 | 数月 | 当天可用（自然语言输入） | 培训成本大幅下降 |
| SAP 导入效率 | 手工 FB01 逐条录入 | 导出 CSV 批量导入 | 一次性完成 |
| 合规审计 | 纸质审批，难追溯 | 全程数字化审批 + 审计日志 | 满足内控要求 |

## 五、扩展性

- **新增业务类型**：在规则表中配置新的科目模板，Agent 自动匹配，无需改代码
- **替换底层大模型**：默认使用 Anthropic Claude，修改 `agents/model_factory.py` 一个文件即可切换为 DeepSeek、通义千问、本地 Ollama 等任意 OpenAI 兼容模型
- **接入企业系统**：审批通知可对接邮件/企业微信/钉钉；导出可改为直接调用 SAP RFC/BAPI，无需 CSV 中间文件
- **多公司代码**：凭证上下文支持 `company_code` 参数，可扩展为多法人实体
